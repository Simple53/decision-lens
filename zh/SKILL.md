# 领域研究与决策支持工作流

## 核心定位

**AI 负责：** 强制调用 `search_web` 提取绝对真实的 URL 避免幻觉、编写**动态高可读性标题**、展开**10 大核心领域要素详解**、全量扫描候选池、**运行 Python 脚本生成可下载的真实 Excel (.xlsx) 文档**、使用**可点击的内联数字角标 `[[1]](URL)`**、在 Phase 1 强制调用系统提问/暂停工具等待确认、编写方案对比矩阵（**无总分、无排名**）。

**核心铁律：严禁做推荐与排名**。AI 只能辅助决策，呈现各维度的得分与情境权衡，**绝对不可以计算“加权总分”并排出 1-2-3 名**。

---

## 铁律

1. **绝对禁止 URL 幻觉 (Zero Hallucination Web Search Protocol)**：
   - AI **必须显式调用 `search_web` 工具** 去查找并验证报告中使用的**每一个** URL。
   - 严禁依靠模型内部记忆（Parametric Memory）凭空捏造或猜测 GitHub 仓库、B站视频或官方文档的 URL。如果搜索不到具体的有效链接，明确说明未找到，绝不伪造根域名敷衍。
2. **可点击的内联数字角标 (Clickable Inline Citations)**：
   - 报告正文中的所有引用角标必须是**真实的、可点击的 Markdown 超链接**，格式必须精确为：`[[1]](https://exact-url.com)`。
   - 严禁仅使用纯文本 `[1]` 或 `[^1]`。
3. **动态高可读性标题 (Dynamic, Topic-Specific Titles)**：
   - 严禁使用诸如“领域研究模板：[主题]”这样僵硬死板的标题。
   - 必须根据用户的具体研究主题，生成**高可读性、引人入胜且贴合主题的大标题**（例如：`# 深入解析 2026 开源桌面 Agent：架构选型与核心矛盾剖析`）。
4. **严禁总分排名与推荐 (NO Rankings & NO Total Scores)**：
   - **本 Skill 仅用于辅助决策，严禁代替用户做决定！**
   - 在“透明加权评分表”中，仅展示各个维度的定性得分（1-5），**绝对不允许计算、汇总或显示“加权总分”列**。
   - 不得为候选方案进行 1, 2, 3 的绝对排名。
   - 将“决策动作建议”板块替换为**“情境权衡分析” (Trade-off Analysis)**，只客观描述：“如果在意 X 因素，方案 A 的 Y 机制更合适”。
5. **10 大核心领域要素详解 (Top 10 Core Domain Factors Breakdown)**：
   - 废除原先简略的金字塔结构/两句话介绍，改为系统性地列出该领域的**约 10 个最核心要素**（涵盖核心机制、技术难点、专有名词、核心张力等）。
   - **内容深度要求**：对于这 10 个核心要素中的每一个，**必须至少撰写一段完整、详细的解释说明**。
6. **真实 Excel (.xlsx) 文档生成规程 (Python Excel Generation)**：
   - 绝不依赖单纯的 Markdown 表格。
   - AI 必须编写并运行 Python 脚本（使用 `pandas` / `openpyxl`），在 Artifacts 目录下生成包含全量候选池与加权评分矩阵的真实可下载 `.xlsx` Excel 文件。
   - 提供可直接点击下载的绝对路径超链接 `[下载全量 Excel 表格](file:///path/to/file.xlsx)`。
7. **无限制候选池扫描与保留 10-15 主力方案**：
   - 输出所有扫描到的候选项目全量清单（标注否决/通过状态及原因）。
   - 从全量候选池中筛选并保留 **10–15 个核心方案** 进入主对比矩阵。
8. **强制交互暂停等待确认 (Mandatory Interactive Pause)**：
   - 在 Phase 1 完成后，AI 必须调用 `ask_question` 工具弹出确认菜单，等待用户回复后再启动后续工作。

---

## 阶段工作流 (Phases)

### Phase 1：研究边界、动态信源探索与参数锁定
明确硬性条件与评估权重；通过 `search_web` 探索发现可靠信源；**强制调用 `ask_question` 工具弹出确认菜单**，等待用户确认。

### Phase 2：核心要素详解、全量池与 Python Excel 生成
撰写动态高可读性标题；深入展开 **10 大核心领域要素**（每条至少一段话解析）；扫描构建全量候选池表；**运行 Python 脚本生成 `.xlsx` Excel 文件**。

### Phase 3：定向深挖与无幻觉网页提取 (`read_url_content`)
在用户批准信源内使用独立 `search_web` 检索验证真实长 URL；调用 `read_url_content` 提取重点项目的真实信息；编写深度技术剖析卡片。

### Phase 4：10-15 核心方案矩阵与定性评分表 (无总分)
构建 10-15 核心方案对比表；构建 1-5 分定性加权评分表（**严禁包含加权总分列**）。

### Phase 5：情境权衡与长 URL 索引
交付客观的情境权衡分析（不推荐、不排名）；附带 Excel 文件下载链接；文末输出使用具体精确长 URL 的 `[1] [标题](https://exact-url)` 参考文献列表。
