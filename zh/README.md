# Decision Lens (领域决策支持 Skill - 中文版 V1.0)

[English Version](../README.md) | [Release v1.0.0](https://github.com/Simple53/decision-lens/releases/tag/v1.0.0)

面向 AI Agent / LLM 的高级 Skill，专注于在陌生或复杂领域建立认知、收集高质量证据、识别核心变量并构建透明可计算的决策矩阵。

> **核心哲学**：Skill 的职责是构建“可验证、可调整、可复算”的决策信息，而不是代替用户输出一个最终答案。

---

## 🌟 V1.0 核心特色

- **决策支持而非顾问推荐**：AI 不输出带有主观倾向的“推荐 A”，而是列出核心变量、证据覆盖率与可调权重矩阵，由用户自主完成取舍。
- **动态信源探索与用户裁决**：拒绝硬编码静态模板。在初筛探索阶段，自动识别针对当前特定问题最靠谱的 3-5 个信源，汇报给用户裁决并可随时删改。
- **硬性条件一票否决 (Hard Constraint Filtering)**：将硬性限制（如“必须是独立桌面端”）作为否决线，初筛时不合格的候选方案直接剔除，决不混入主推荐矩阵。
- **角标/尾注引用系统 (Endnote Citation System)**：正文中所有关键结论、项目特征、指标均附带角标 `[^1]`、`[^2]`，文末输出完整的 `## 参考文献与索引`。
- **透明化权重矩阵**：将决策变量拆解，明确标注权重来源（用户明确要求 / 场景推导 / 默认权重）。用户只需调整权重即可快速重新计算分值。
- **深度技术剖析卡片 (Deep-Dive Cards)**：为通过初筛的主方案提供包含 API 配置（DeepSeek/SiliconFlow/Ollama）与多 Agent 架构机制的深度剖析。

## 🚀 安装与使用

在 Antigravity 全局配置目录（`~/.gemini/config/skills.json`）中注册路径：

```json
{
  "entries": [
    { "path": "path/to/decision-lens" }
  ]
}
```

## 🛠 工作流阶段 (Workflow)

1. **Phase 1: 研究边界、动态信源探索与参数锁定** - 明确目标类型、硬性限制条件、动态探索发现的信源平台列表与评估权重，由用户确认锁定。
2. **Phase 2: 候选池扫描与领域地图** - 构建 6-8 个候选池、执行硬性条件过滤、建立知识结构与核心张力模型。
3. **Phase 3: 定向深挖与冲突分析** - 在用户批复的信源范围内定向检索，整理冲突观点并编写深度技术剖析卡片。
4. **Phase 4: 方案矩阵与加权评分** - 输出结构化对比矩阵，生成带权重透明度的定性打分表（1-5分）。
5. **Phase 5: 决策交付与尾注索引** - 交付已知事实、待验证项、建议验证路径、不同假设下的决策路径提示以及完整的 `## 参考文献与索引` 尾注列表。
